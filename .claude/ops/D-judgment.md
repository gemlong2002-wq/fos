# D. 判斷力外化：弱模型可執行的判準

> 讀者：Sonnet 級以下的主對話模型。每節格式：判準 → 正例 → 反例。
> 用法：遇到標題描述的情境時查對應節，逐條比對，不憑感覺。

## 1. 何時該升級模型

**升級，若命中任一**：
- 同一子任務已失敗 2 次（含），且失敗原因不是缺資料或打錯字這類可自修的錯。
- 任務涉及跨 ≥3 個模組的因果推理（改 A 會影響 B 再影響 C）。
- 需要在互斥方案間做架構級取捨，且選錯的回頭成本 > 半天工作量。
- 除錯時已確認「症狀」但 2 輪後仍提不出可檢驗的「病因假說」。

**不升級，若**：任務量大但同質（升級解決不了量的問題，拆小批次才對）；或失敗原因已明確是缺輸入資料（先補資料）。

- 正例：Sonnet 修一個 race condition，兩輪修改後測試仍間歇性失敗，且說不出失敗的觸發序列 → 帶兩輪的 diff 與測試 log 升 Opus。✔
- 反例：要把 200 個檔案的 import 風格統一，Sonnet 覺得「量大很難」想升 Opus → 錯。這是同質批次活，正解是 Sonnet 解出模式後派 Haiku 分批套。✘

## 2. 何時算真的完成

**「完成」= 以下全部為真**，缺一即未完成，不得回報完成：
1. 驗收條件逐條打勾，每條有證據（測試輸出、read-back 結果、檔案:行號）。
2. 驗收由 fresh-context agent 執行過（`C-dispatch.md` §6），不是執行者自己說的。
3. 產物已落檔且 read-back 確認存在、完整（尤其：宣稱寫了檔 ≠ 檔真的在）。
4. 沒有留下「暫時先這樣」的未註記妥協；有妥協就寫進回報，由使用者決定接不接受。

- 正例：實作完成，`pytest` 全綠（貼統計），fresh agent read-back 確認新檔存在且含驗收要求的三個函式，回報附一條已知限制「未處理 unicode 檔名」。✔
- 反例：「我已完成修改並確認邏輯正確」——沒有測試輸出、沒有 read-back、「確認邏輯正確」是自驗。這句話出現即視為未完成。✘

## 3. 何時該停下來問使用者

**停下來問，若命中任一**：
- 關鍵變數缺失且存在**互斥解讀**（選 A 和選 B 會產出不相容的結果）。
- 動作不可逆或影響範圍出圈：刪資料、對外發送、改動使用者沒提到的系統、花錢。
- 完成任務會違反使用者偏好（限本 session 使用者明確說過、或 CLAUDE.md/lessons.md 已登記者）或制度鐵律。
- 發現任務的前提可能是錯的（使用者要 X 是為了 Y，但 X 其實達不成 Y）。僅當有具體技術性反例證明 X 達不成 Y 時才觸發停下問；否則標【推】選最合理解讀續推。

**不要問，若**：只是細節缺失而主軸明確 → 標【推】選最合理解讀直接推進，在回報中列出所做假設。問使用者是最貴的操作，批次問、附選項與建議，不要擠牙膏式一次問一題。

- 正例：「把舊 API 下線」——下線 = 標記 deprecated 還是直接刪 endpoint？兩者結果互斥且刪除不可逆 → 停下問，附兩案的影響比較。✔
- 反例：寫報告時不確定日期格式用 ISO 還是斜線 → 跑去問。錯。標【推】選 ISO 推進，回報時註記即可。✘

## 4. 方向錯了的訊號（該換路，而非重試）

**命中任一即停止當前路線**，換路 = 改變方法或核心假設：
- 連續 2 輪修改後，錯誤訊息**沒有變化**（同樣的錯 = 你修的不是病因）。
- 為了讓方案成立，需要的「例外處理/特判」數量在增加而非減少。
- 每前進一步都要再多繞一個新障礙（工具不支援 → 找替代 → 替代又缺功能 →…連續 3 層即命中）。
- 回頭重讀任務目標，發現當前工作與驗收條件的連結已說不清楚。

- 正例：為了讓套件 X 在環境跑起來，已依序處理版本衝突 → 缺系統依賴 → 依賴又要求升級 Python。三層障礙 → 停，換路：找不依賴 X 的實作方式，或問使用者能否換環境。✔
- 反例：測試失敗，把同一段程式碼換個寫法再跑，錯誤訊息一模一樣，再換個寫法又跑——這是換措辭不是換路。正解：先寫出病因假說並設計能區分假說的最小實驗。✘

## 5. 品質底線怎麼驗

任何產出交付前的最低門檻（依類型取用）：

**程式碼**：有測試就全跑；沒測試就為改動處寫最小測試再跑；lint/type check 通過；diff 範圍不超出任務宣告的檔案清單。
**文件/報告**：BLUF 在最前；每個事實主張標【實/推/臆】且【臆】附升級路徑；引用的檔案路徑與指令全部實際驗證過存在/可跑（用小模型逐一 read-back/dry-run）。
**制度/設定檔修改**：改前有備份；改後 read-back；與既有規則無衝突（grep 相關關鍵字比對）。

- 正例：報告引用了 5 個檔案路徑，交付前派 Haiku 逐一 `ls` 確認，發現 1 個路徑打錯，修正後交付。✔
- 反例：報告寫「執行 `npm run migrate` 即可」，但該 script 是否存在沒查過就交付 → 使用者跑了報錯。路徑與指令未經驗證即交付 = 品質底線未過。✘

## 6. 本檔的極限

以上判準覆蓋「可規則化的判斷」。以下情況規則救不了，直接走 `C-dispatch.md` §7：
美感與品味、政治與人際敏感度、使用者沒說出口的真實意圖。訊號：你發現自己在判準之間「都對又都不對」地打轉 → 停止套規則，升級或問人。
